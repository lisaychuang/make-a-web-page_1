Session 0: The Web and Basic HTML

Basics of the Web

The World Wide Web (WWW) is a collection of HTML documents:

HTML = HyperText Markup Language, used to tell computers what to do / display on webpages. HTML codes documents, which form the heart of the web:

  • Text Content = what you see
  • Mark Up = look of content/ arrangement, can be stylized using CSS)
  • Reference to other documents (e.g. images, videos)
  • Links to other pages using Hyperlinks
    * Hyperlinks were invented in 1990s.

In another word, coding is when programmers (or you!) write text in instructional language that the computer understands.

On our computer, we use web browsers (e.g. Chrome, IE) to make request to servers - a collection of computers sitting in a room storing files and data, using HTTP. HTTP are the protocols browsers used to communicate with servers.

Computers are stupid

Computers are sensitive to typos, and small mistakes such as missing an End Tag will affect the page layout! Developers must then "debug" to track and correct their mistakes.

HTML Markup

Markups are made up of "Tags", which looks like this:
< Tag > Contents </ Tag >

First tag = Opening Tag, which gives instruction to how the content should be arranged.
After the content, we will add a Closing tag, to signal the end of the instruction (eg end of Bold/Italic)
This structure is referred to as a HTML Element.

We can Italicize text using < em >

HTML Attributess

An attribute is a property of an HTML element. It looks like this:
< TagName ATTR = "value" > Contents < /TagName >

An example of an attribute is the ANCHOR = < a > tag.
To create a link to an external page, use:

< a href = "link" > : href = attribute name , and the link = URL to link to.

We can Create a link to Andy's Codepen examples using this tag.

< img src = "link" alt = "text" > : img = images , src = source, link = URL to source image from, alt = alternative display, text = alternative text shown when image doesn't load, which is useful for broken request and blind people! :)

NB: These tags are all inline elements

White spaces: in-line v.s. blocks

When coding a web document using HTML, it ignores white spaces (eg tab, multiple spaces etc). Useful HTML tags to create line breaks are:

< br > : single inline break, which ends a line and wrap to the next line. It is a "void" tag which doesn't require a closing tag < /br >
< p > : Whereas the P tag Creates a block paragraph, creating an invisible box around the text... it can have height & width. P tag will also require a closing tag.

Container Elements: Span & Div

Both these elements contain content, and can have style attached to them via CSS to change the way the text will look/ their layout in the document.
< span > : This is what a SPAN line looks like
* SPAN creates an inline break, similar to the BR tag

This is what a DIV line looks like

* DIV creates another block, similar to the P tag for paragraph text.

HTML Documents

A complete HTML document has a DOC type, opening & closing HTML tag, the HEAD (includes title of the document, meta data / CSS / Javascript), and Body Tag = which includes most of the content in the document.
In the Nanodegree course, we will be focusing on the BODY tags to generate content of a HTML document.

Session 1: HTML, CSS, Editors and Structure

Creating Structured Documents

HTML, CSS, Javascript : 3 langugages serving different purposes when developing a webpage.

Using the analogy of a house...

  • HTML: HyperText Markup Language = the standard markup language used to create web pages. HTML creates structure in a webpage (think layout of a house)
  • CSS: Cascading Style Sheets = language used for describing the look and formatting of a webpage. CSS creates style in a webpage, such as font size, colours, thickness of borders etc (think decoration within the house)
  • Javascript = language used to create interactive components in a webpage (think of remotes or garage doors needed to perform actions)

Using Google Chrome Developer Tools

Click on Google Chrome hamburger sign > Tools > Developer's Tools, a box opens up on the bottom panel.

On review, you can see that :

  • All HTML elements are rectangular
  • By going deep enough into the code, you can read the same text as on the webpage.

    HTML builds a Tree-like structure for the webpage. This is how HTML classifies page content, telling the browser which part is text/ image / link etc.

Document Object Model (DOM)

Document Object Model: a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

When programmers talk about the DOM, they are talking about the tree-like structure of a webpage.
DOM converts an HTML tag into an element that the browser uses to create a page.

CSS Styling Summary

CSS is code written to define the "style" of HTML elements in a webpage.
A great resource to learn new CSS properties is: Mozilla CSS reference page

Use CSS Class selector to define style of classes, which in turn changes the styling of corresponding HTML elements.

It is also crucial to avoid repetition in your HTML structure. When you repeat codes over and over, you are more likely to make errors - remember, computers are stupid and a small typo will affect your code!. By avoiding repetition, you can saves time when it comes to DeBugging! In addition, an organized structure (e.g. using DIV and SPAN class attribution) will enable you to apply style to HTML element with a single CSS line.
This is why you should build up the habits to apply Procedural Thinking as you learning to code, and avoid repetition!

Using HTML to "Boxify"

Everything on a webpage is organized in "Boxes" - or rectangular sections. This gives developers the flexibility to rearrange content easily on a webpage, and helps them "Boxify, or break down a given graphic design into various boxes.

When coding a webpage, you can build the structure of the page using Div and Span elements. Use DIV to boxify a section, and use SPAN to add style to in-line elements.
Within each element, you can include Class Attributes to help you label each box.

Box Sizing

Carrying on from previous point, each HTML "box" has 4 elements: margin, border, padding, content. Refer to this demo page for more details.

With 4 components to each box, it can often be hard to get the accurate size of a box. Use two techniques to deal with sizes:

  1. Set sizes in terms of percentages rather than pixels.
  2. Set the box-sizing attribute to border-box for every element.

Use this CSS code to ensure your box sizing is displayed accurately in all browsers (Safari, Mozilla, Microsoft IE)

* {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

Box Positioning

As Divs are block elements, they take up the entire width of a page. Adding the CSS rule:
display: flex;
to override this behavior and let divs appear next to each other.

You can read more about Flexible Boxes here.

Code, Test, Refine

Here are steps to CODE your webpage:

  1. Look for natural boxes: Build structure of the webpage by visually boxifying the given design
  2. Look for repeated styles & semantic elements: Write your HTML, and add relevant headings
  3. Apply CSS styles: Consider box sizing and positioning. Go from BIG to SMALL.
  4. Polish finer details: fix HTML & CSS as needed.

Remember you can always use the Developer Tool function to test HTML & CSS with instant feedback on the webpage, before you change your code on source file.

Other Notes

To ensure the body element / content of your webpage changes as size of browser changes, you can define div element with:
width: 75%
This ensures the width of the content of the page is always shown at 75% of the browser screen width.

Remember to review your HTML code, to ensure sound structure is in place - differentiating between DIV and SPAN tags, for example.
